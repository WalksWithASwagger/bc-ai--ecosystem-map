# BC AI Ecosystem Atlas - Templates

*Reusable templates and examples for data management*

## 📋 Available Templates

### 🔄 Batch Update Template (`sample-updates.json`)
**Template for batch updating organization information**

```json
[
  {
    "pageId": "notion-page-id-here",
    "updates": {
      "Website": { "url": "https://example.com" },
      "LinkedIn": { "url": "https://linkedin.com/company/example" },
      "Email": { "email": "contact@example.com" },
      "Phone": { "phone_number": "+1 604 555 0123" }
    }
  }
]
```

**Usage:**
```bash
node tools/utility/batch-update.js templates/sample-updates.json
```

## 🛠️ Using Templates

### 1. Organization Import
```bash
# Copy template and customize
cp templates/organization-template.md data/discoveries/2025-08-01_new-batch.md

# Edit the file with your discoveries
# Import when ready
node tools/import/import-discovery-orgs.js data/discoveries/2025-08-01_new-batch.md
```

### 2. Batch Updates
```bash
# Copy template and customize
cp templates/sample-updates.json my-updates.json

# Edit with your page IDs and updates
# Apply when ready
node tools/utility/batch-update.js my-updates.json
```

### 3. Research Documentation
```bash
# Copy template for new research session
cp templates/research-template.md data/research/2025-08-01_cleantech-research.md

# Document your research findings
# Convert to discovery file when complete
```

## 📋 Best Practices

### File Naming
- **Date prefix**: Always start with YYYY-MM-DD
- **Descriptive names**: Include topic or focus area
- **Consistent format**: Follow established patterns

### Data Quality
- **Source attribution**: Always document where information came from
- **Verification**: Cross-check critical information
- **Completeness**: Aim for all required fields
- **Accuracy**: Prioritize current, verified information

### Documentation
- **Clear descriptions**: Make purpose and scope obvious
- **Update regularly**: Keep templates current with process changes
- **Version control**: Track changes to important templates
- **Examples**: Include real-world usage examples
